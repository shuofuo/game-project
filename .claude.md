## 🔧 调试规则

### 每次改代码前必须：
1. 先用 Playwright 量当前数据（offsetWidth/offsetLeft/offsetTop）
2. 明确说出改动前后的数据对比
3. 用户确认后再执行

### 改完后必须：
1. Playwright 重新量数据
2. 截图（save_full_page=False）
3. 对比改动前后数据，确认问题是否解决
4. **未解决不继续，问用户下一步方向**

## ⌨️ 开发流程规则

### 标准流程：
```
1. 用户描述问题或目标
2. 我量数据 → 说清楚当前状态
3. 我说出修改计划 → 用户确认
4. 我执行修改 → 验证数据
5. 用户看效果 → 反馈
6. 循环 1-5
```

### 批量改动规则：
- 每次最多改 2 个相关小改动（不多改）
- 改了 A 发现 B 也有问题 → 停下，问用户是先修 B 还是继续 A
- 不在一次对话里做多个独立功能（分开会话）

### 改代码前必问：
- "我计划改 X，我确认一下：你想现在就修这个，还是先做别的？"

## 📐 布局调试 Playwright 模板

```python
r = pg.evaluate('''() => {
  const mr=document.getElementById('middleRow');
  const hc=document.getElementById('heroCard');
  if(!mr||!hc) return{error:'missing'};
  const mrW=mr.offsetWidth;
  const hcW=hc.offsetWidth;
  const hcLeft=hc.offsetLeft;
  const gapL=hcLeft;
  const gapR=mrW-hcLeft-hcW;
  return {
    heroCard:{w:hcW,h:hc.offsetHeight,left:hcLeft,leftGap:gapL,rightGap:gapR},
    isCentered:Math.abs(gapL-gapR)<10,
    children:Array.from(hc.children).map((c,i)=>({
      i,text:c.textContent.trim().slice(0,25),
      left:c.offsetLeft,w:c.offsetWidth,
      right:hcW-c.offsetLeft-c.offsetWidth
    }))
  };
}''')
pg.screenshot(path='screenshots/stepN.png',full_page=False)
```

## 💰 省 Token 技巧

1. **不说废话** - 直接说数据和计划，不说"好的我来帮你"之类的
2. **用户说"继续"不等于"继续下一步"** - 可能用户只是看日志
3. **主动说"这个改动可以一次说清楚，你确认吗"**
4. **不要重复已经验证过的数据** - 数据只说一次
5. **有疑问先问用户** - 不确定方向时问，不要瞎猜
